CheeseToken Whitepaper

A Gamified Token on Shibarium

Introduction

CheeseToken (CHEESE) is a unique ERC20 token deployed on the Shibarium blockchain, designed to bring an exciting "catch" game to the Shiba Inu ecosystem. Leveraging Shibarium's fast 5-second block time and BONE as the native gas token, CheeseToken introduces a decentralized, gamified experience where players can steal tokens from each other by transferring a fixed amount of CHEESE. This whitepaper details the token's specifications, rules, and mechanics.

Token Specifications

  • Name: Cheese
  • Symbol: CHEESE
  • Decimals: 18
  • Total Supply: 1,000,000,000,000,000 CHEESE (1 quadrillion)
  • Initial Distribution: All tokens are minted to the deployer upon deployment.
  • Blockchain: Shibarium
  • Block Time: 5 seconds
  • Native Currency: BONE

Game Rules

CheeseToken's core feature is a "catch" game where players can steal tokens from others. Below are the specific rules governing this mechanic:

1. Roles

Roles are assigned based on the NoneToken contract at 0x64E19b6a167878d2483d166212A3c2c68b1eB842:

  • Dog: Address where noneToken.isNode(address) == true.
  • Cat: Address where noneToken.isNode(address) == false and noneToken.hasPaidBone(address) == true.
  • Rat: Address where neither condition is met.
  • Query: Use getRole(address) to retrieve an address's role ("Dog", "Cat", or "Rat").

Note: Role status is dynamic and updated in real-time via NoneToken.

2. Catching Mechanism

  • Trigger: Transfer exactly 1 CHEESE (1 * 10^18 wei) to a target address using transfer(address to, uint256 amount).
  • Cost: Catcher pays 1 CHEESE, requiring a minimum balance of 1 CHEESE.
  • Reward: Catcher receives 1% of the target's balance (_balances[target] / 100).
  • Minimum Reward: Reward must be at least 1 wei; targets with less than 100 wei cannot be caught.

3. Catching Conditions

  • Game Status: Must be active (isGameActive == true), controlled by the owner.
  • Catcher Cooldown: Catchers must wait 20 hours (72,000 seconds) between catches, tracked by lastCatchTime[catcher].
  • No Target Cooldown: Targets can be caught repeatedly with no delay.
  • Catcher Eligibility: Only Dogs and Cats can catch; Rats are ineligible.
  • Target Restrictions:
    • Must not be a Dog (noneToken.isNode(target) == false).
    • Must not be whitelisted (whitelist[target] == false).
    • If catcher is a Cat, target must not be a Cat (Cats can only catch Rats).

4. Catch Execution

  • Catcher transfers 1 CHEESE to the target.
  • Target transfers 1% of its balance to the catcher.
  • Catcher's cooldown updates to lastCatchTime[catcher] = block.timestamp.
  • Catch is logged in catchRecords[target] with catcher address, amount, and timestamp.

5. Whitelist

  • Purpose: Protects addresses from being caught.
  • Management: Owner can add (addToWhitelist(address)) or remove (removeFromWhitelist(address)) addresses.

6. Catch Records

  • Storage: Recorded in catchRecords[target].
  • Structure: Includes catcher (address), amount (uint256), timestamp (uint256).
  • Query:
    • getCatchRecords(address target, uint256 page, uint256 pageSize): Retrieves paginated records.
    • getCatchRecordsCount(address target): Returns total catch count for a target.

7. Game Control

  • Owner Functions:
    • stopGame(): Disables catching (isGameActive = false).
    • startGame(): Enables catching (isGameActive = true).
    • renounceOwnership(): Sets owner = 0x0, permanently relinquishing control.
  • Default: Game starts active.

Token Functions

CheeseToken implements the ERC20 standard with the following key functions:

  • transfer(address to, uint256 amount): Transfers tokens; triggers catching if amount == 1 * 10^18.
  • approve(address spender, uint256 amount): Approves a spender.
  • transferFrom(address from, address to, uint256 amount): Transfers on behalf of another (no catching).
  • increaseAllowance(address spender, uint256 addedValue): Increases allowance.
  • decreaseAllowance(address spender, uint256 subtractedValue): Decreases allowance.
  • balanceOf(address account): Returns balance.
  • allowance(address owner, address spender): Returns allowance.

Shibarium Integration

  • Fast Transactions: Shibarium’s 5-second block time ensures quick catch confirmations.
  • BONE Gas Fees: All transactions, including catches, use BONE as the native currency.
  • Ecosystem Fit: Ties into the Shiba Inu ecosystem via NoneToken roles.

Security and Design

  • Solidity: ^0.8.0 with overflow protection.
  • Fixed Supply: No minting or burning; total supply is constant.
  • Dynamic Roles: Real-time role checks via NoneToken.
  • Game Balance: 20-hour catcher cooldown vs. no target cooldown encourages active play.

Conclusion

CheeseToken brings a fresh, gamified twist to the Shibarium blockchain, combining tokenomics with a thrilling catch mechanic.
