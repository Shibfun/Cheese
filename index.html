CheeseToken Whitepaper

A Deflationary Catch Game on Shibarium

Introduction

CheeseToken (CHEESE) is a gamified ERC20 token on the Shibarium blockchain, blending tokenomics with a unique "catch" mechanic. Players can steal tokens from others while contributing to a deflationary supply through burning. Built for the Shiba Inu ecosystem, CheeseToken leverages Shibarium's 5-second block time and BONE as the native gas token to deliver a fast, engaging experience. This whitepaper outlines the token's mechanics and rules.

Game Rules

CheeseToken's core is a "catch" game with a deflationary twist. Below are the detailed rules:

1. Roles

Roles are determined by the NoneToken contract at 0x64E19b6a167878d2483d166212A3c2c68b1eB842:

  • Dog: noneToken.isNode(address) == true
  • Cat: noneToken.isNode(address) == false and noneToken.hasPaidBone(address) == true
  • Rat: Neither condition met
  • Query: getRole(address) returns "Dog", "Cat", or "Rat"

Roles update dynamically via NoneToken.

2. Catching Mechanism

  • Trigger: Transfer 1 CHEESE (1 * 10^18 wei) via transfer(address to, uint256 amount).
  • Cost: Catcher pays 1 CHEESE.
  • Steal Amount: 1% of target's balance (_balances[target] / 100).
  • Catcher Reward: 0.7% of target's balance (70% of stolen amount).
  • Burn Amount: 0.3% of target's balance (30% of stolen amount, burned to reduce total supply).
  • Minimum Steal: Stolen amount must be ≥ 1 wei (target balance ≥ 100 wei).

3. Catching Conditions

  • Game Status: isGameActive == true, owner-controlled.
  • Catcher Cooldown: 20 hours (72,000 seconds) between catches, tracked by lastCatchTime[catcher].
  • No Target Cooldown: Targets can be caught repeatedly.
  • Catcher Eligibility: Dogs and Cats only.
  • Target Restrictions:
    • Not a Dog (noneToken.isNode(target) == false).
    • Not whitelisted (whitelist[target] == false).
    • If catcher is a Cat, target must not be a Cat.

4. Catch Execution

1Catcher transfers 1 CHEESE to target.

5. Whitelist

  • Purpose: Prevents catching.
  • Management: Owner uses addToWhitelist(address) and removeFromWhitelist(address).

6. Catch Records

  • Storage: catchRecords[target].
  • Structure: catcher, amount (catcher reward), timestamp.
  • Query:
    • getCatchRecords(address target, uint256 page, uint256 pageSize)
    • getCatchRecordsCount(address target)

7. Game Control

  • Owner Functions:
    • stopGame(): isGameActive = false
    • startGame(): isGameActive = true
    • renounceOwnership(): owner = 0x0
  • Default: Game starts active.

Token Functions

  • transfer(address to, uint256 amount): Transfers tokens; triggers catch if amount == 1 * 10^18.
  • approve(address spender, uint256 amount)
  • transferFrom(address from, address to, uint256 amount): No catching.
  • increaseAllowance(address spender, uint256 addedValue)
  • decreaseAllowance(address spender, uint256 subtractedValue)
  • balanceOf(address account)
  • allowance(address owner, address spender)
